# 360-Degree Audit ‚Äî 2026-07-10

Full-project review of **python-hvlp** (Hilscher Variable Length Protocol ‚Äî an
MQTT-inspired publish/subscribe protocol over TCP, used as **training material**
for OOP, SOLID, sockets and threading). Requested as a deep code review +
structure audit + 360-degree analysis, with **trainer** and **trainee** as the
primary stakeholders.

Run per `docs/solid-ai-templates/templates/base/workflow/360.md`. Because HVLP
is a **headless library/CLI** with no user-facing web surface, the ¬ß360-headless
adaptation applies: the mechanics are kept (seven parallel, context-isolated
reviewers, each with a clean context and its own role prompt; adversarial
per-finding verification; letter grades; a persisted dated report), and the
default four perspectives are re-projected onto the two named stakeholders
(**Trainee**, **Trainer**) plus five engineering dimensions. The three requested
deliverables map onto these reviewers:

- **Deep code review** ‚Üí *Correctness* dimension
- **Structure audit** ‚Üí *Architecture & Structure* dimension
- **360 analysis** ‚Üí all seven perspectives merged (this document)

**Overall grade = the lowest dimension grade.**

### Independent verification (parent agent)

The crux runtime findings were reproduced by the parent agent before grading,
against the real modules on **Python 3.13.0** (`six 1.17`, `pytest 9.1`):
`set.unsubscribe` `AttributeError` on the disconnect/dead-subscriber paths;
`ValueError` on payloads > 253 bytes; truncated-frame desync; `UnicodeDecodeError`
on an invalid-UTF-8 topic escaping `HvlpParsingError`; and `get_subscribers`
returning the live internal set. The shipped `venv/` is dead (its base
interpreter `C:\BuildTools\python\3.12.2` is absent); a clean interpreter was
used for all runs. No project files were modified during the audit.

either crash or *misteach*.

## Current bottleneck

**Correctness (D)** ‚Äî a single root-cause defect is the highest-leverage fix:
`hvlp/session.py:179` and `:303` call `.unsubscribe()` on `self.register[topic]`,
which is a plain `set` (built in `register.py:81`), not the register. A `set` has
no `unsubscribe` method, so **every subscribed client that disconnects ‚Äî the most
ordinary lifecycle event ‚Äî throws an uncaught `AttributeError` that kills the
session thread and leaks the subscription** (verified: register still shows
`{'test': 1}` after a clean disconnect). The same bug re-fires on the
dead-subscriber publish path, crashing the *publisher's* thread. This one defect
also underlies Security #7 and Trainer #5.

**Testing (D) is the reason it stayed hidden**: the shipped suite cannot be
collected (`test_broker.py` runs a live broker at import), `test_session.py`
fails and hangs the interpreter on leaked non-daemon threads, and the four
runtime-critical modules (client, broker, session, register) have **0% working
coverage** ‚Äî so none of the seven Critical/High runtime bugs are exercised.
Fixing Correctness without fixing Testing will not stop the next regression.
